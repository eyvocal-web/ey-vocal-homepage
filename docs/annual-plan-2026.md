# EY보컬팀 1년 프로젝트 계획서

기간: 2026.04 ~ 2027.03

## 1. 프로젝트 개요

EY보컬스튜디오 홈페이지를 정식 런칭하고, 검색 유입과 상담 전환을 안정적으로 운영하기 위한 1년 계획입니다.

- 프로젝트 범위: 홈페이지 정식 배포, 검색 유입 기반 구축, 콘텐츠 운영, 상담 전환 개선
- 운영 원칙: 정적 HTML/CSS/JS 유지, 외부 의존성 최소화, 카카오톡·전화 중심 상담 구조 유지
- 협업 전제: 콘텐츠 확정은 성표님 판단, 구현과 QA는 ey보컬팀 기준으로 진행

## 2. 현재 상태

### 현재 구현 상태

- 단일 페이지 홈페이지 구성 완료
  - Hero, 소개, 강사, 레슨 프로그램, 후기, FAQ, 시설 안내, 블로그 프리뷰, 오시는 길, 상담 CTA
- 프로그램 섹션은 `취미 보컬 레슨` 단일 운영 기준으로 반영됨
- 구조화 데이터 반영 완료
  - `LocalBusiness`
  - `FAQPage`
- FAQ 섹션과 블로그 프리뷰 섹션이 이미 포함됨
- 스튜디오 이미지는 JPG + WebP fallback 구조로 정리됨
- Pretendard 서브셋 폰트를 self-host로 운영 중

### 기술 상태

- 기술 스택: HTML5, CSS3, Vanilla JavaScript
- 외부 프레임워크/런타임 의존성: 없음
- 배포 대상: 정적 호스팅(Vercel 중심)
- 폰트: `assets/fonts/PretendardVariable.subset.woff2` 89KB

### 당장 남은 작업

- `codex/init` 브랜치의 변경분 정리 및 커밋
- `main` 머지와 실제 배포 반영
- 최종 QA 체크리스트 재확인
- 지도 시각 자료 여부 최종 판단

## 5. 주요 리스크

### 1. 콘텐츠 확정 지연

후기, 블로그 글, 지도 시각 자료, 촬영 자산이 늦어지면 Q2~Q3 일정이 밀릴 수 있습니다.

대응:
- 콘텐츠가 없어도 운영 가능한 기본 구조를 유지
- 실제 콘텐츠 확보 시 교체만으로 반영 가능하게 설계 유지

### 2. 측정 기반 부재

현재 저장소에는 analytics 도입이 없어 전환 개선 판단이 어렵습니다.

대응:
- Q2 초에 추적 기반 구축을 선행 마일스톤으로 분리
- Q3의 CTA 비교와 퍼널 분석은 추적 도입 이후에만 진행

### 3. 운영 범위 확대 리스크

예약, 다국어, 외부 시스템 연동은 현재 정적 사이트 운영 원칙과 충돌할 수 있습니다.

대응:
- Q4에서는 구현이 아니라 검토만 수행
- 원칙 충돌 시 2차년도 과제로 이관

### 4. 배포 전 마감 지연

현재 변경분이 `codex/init` 브랜치에 남아 있어 머지와 배포가 지연될 수 있습니다.

대응:
- Q1 첫 마일스톤을 커밋 정리와 배포 완료에 집중
- 배포 후에만 SEO·콘텐츠 확장 일정 진행
