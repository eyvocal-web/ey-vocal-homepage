# EY보컬스튜디오 네이버플레이스 후기 연동 조사

대상 프로젝트: `ey-vocal-homepage`

현재 기준:
- `#testimonials` 섹션 후기 4건은 모두 placeholder입니다.
- 정적 사이트 구조라 배포 시점에 확정 데이터가 있어야 가장 안정적입니다.
- 네이버 공식 오픈 API 목록에서는 플레이스 리뷰 전용 API를 확인하지 못했습니다.

## 1. 비교 기준

- 가용성
- 약관 및 저작권 리스크
- 갱신 자동화 난이도
- 구현 호흡
- 현재 다크 테마와의 디자인 일치
- 운영 부담

## 2. 옵션 비교

| 옵션 | 가용성 | 약관 리스크 | 갱신 방식 | 구현 호흡 | 디자인 일치 | 운영 부담 |
|---|---|---|---|---|---|---|
| 네이버 공식 API/위젯 | 낮음 | 낮음 | 자동 가능 | 현재 확인 불가 | 중간 | 낮음 |
| iframe/외부 임베드 | 낮음 | 중간 | 자동 | 낮음 | 낮음 | 낮음 |
| 스크래퍼 + JSON 캐시 | 중간 | 높음 | 자동 | 높음 | 높음 | 높음 |
| 수동 export → JSON 갱신 | 높음 | 낮음 | 수동 | 낮음 | 높음 | 중간 |
| 자체 후기 폼 | 높음 | 낮음 | 반자동 | 중간 | 높음 | 중간 |

핵심 판단:
- 공식 API/위젯은 현재 공개 문서 기준으로 바로 연결 가능한 경로를 확인하지 못했습니다.
- 스크래퍼는 약관, 저작권, 차단 리스크가 커서 운영 안정성이 낮습니다.
- 정적 사이트와 현재 운영 리소스를 기준으로는 수동 JSON 갱신이 가장 안전합니다.

## 3. 권장안 1순위

### 수동 export → JSON 갱신

권장 이유:
- 법적·운영 리스크가 가장 낮습니다.
- 정적 사이트 빌드 흐름에 자연스럽게 맞습니다.
- 디자인을 현재 testimonials 섹션에 그대로 유지할 수 있습니다.

구현 흐름:
1. 네이버플레이스에서 노출 가능한 후기만 운영자가 수동 선별합니다.
2. 후기를 `data/reviews.json`에 정리합니다.
3. 정적 빌드 또는 배포 전 스크립트에서 HTML에 주입합니다.
4. placeholder 4건은 실제 후기 데이터로 교체합니다.

권장 운영 주기:
- 월 1회 정기 갱신
- 신규 후기 누적 시 수시 반영 가능

## 4. 권장안 2순위

### 자체 후기 폼 + 수동 승인

권장 이유:
- 네이버 외부 채널 의존도를 낮출 수 있습니다.
- 후기 형식과 톤을 일관되게 관리할 수 있습니다.

주의:
- 수집 동의 문구와 공개 기준을 별도 마련해야 합니다.
- 초기에는 폼, 저장소, 승인 프로세스까지 추가 설계가 필요합니다.

## 5. 비권장안

### 스크래퍼 + 자동 수집

비권장 이유:
- 네이버 API/서비스 약관과 충돌할 여지가 큽니다.
- 자동화 접근이나 주기적 수집은 차단 리스크가 있습니다.
- 수집 결과를 별도 DB로 캐시·가공하는 운영 방식도 부담이 큽니다.

### iframe/임베드

비권장 이유:
- 현재 공개 문서 기준으로 안정적인 공식 후기 위젯 경로를 확인하지 못했습니다.
- 반응형, 다크 테마, 레이아웃 통제가 어렵습니다.

## 6. 권장 데이터 모델

파일 경로 제안:
- `data/reviews.json`

예시 스키마:

```json
[
  {
    "id": "review-001",
    "source": "naver-place",
    "nameMask": "김○○",
    "course": "취미 보컬 레슨",
    "period": "3개월",
    "summary": "높은 음에서 힘이 덜 들어가고 복습 포인트가 분명해졌습니다.",
    "body": [
      "수업 후 연습 방향을 바로 정리할 수 있어서 혼자 연습할 때도 덜 막혔습니다.",
      "무리하게 음을 올리기보다 목소리에 맞는 순서로 잡아 주셔서 편했습니다."
    ],
    "rating": 5,
    "publishedAt": "2026-04-01",
    "isApproved": true
  }
]
```

필수 필드:
- `id`
- `source`
- `nameMask`
- `course`
- `period`
- `summary`
- `body`
- `publishedAt`
- `isApproved`

## 7. 운영 메모

- 후기 원문을 그대로 복사하기보다, 공개 허용 범위와 저작권 이슈를 먼저 확인합니다.
- 네이버 후기를 바탕으로 하더라도 사이트 게시용 문안은 최소 편집 기준을 정해 두는 편이 안전합니다.
- 별점, 작성일, 수강 기간은 사이트 신뢰도에 도움이 되지만 과장 표현은 피합니다.

## 8. 현재 권장 결론

- 1순위: 수동 export → `data/reviews.json` 갱신
- 2순위: 자체 후기 폼 + 수동 승인

구현 호흡 1줄:
- 지금 단계에서는 수동 JSON 방식으로 먼저 운영하고, 후기 수가 안정적으로 쌓이면 자체 수집 폼을 붙이는 순서가 가장 현실적입니다.
