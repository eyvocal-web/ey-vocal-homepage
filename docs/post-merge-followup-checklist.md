# EY보컬스튜디오 Post-Merge 후속 체크리스트

대상 프로젝트: `ey-vocal-homepage`

기준 시점:
- local `main`은 `5419972`까지 머지된 상태입니다.
- production push는 GitHub 인증 차단으로 아직 미완료입니다.
- 본 문서는 push 차단 해소 후 다음 PR에서 바로 반영할 후보만 정리합니다.

현재 기준점:
- Lighthouse 모바일: `90 / 100 / 96 / 100`
- Lighthouse 데스크톱: `100 / 100 / 96 / 100`
- `html-validate`, `node --check js/main.js` 통과

범위 메모:
- Hero는 CSS 배경 사진 구조이므로 `alt` 점검 대상에서 제외합니다.
- 이미지 대체 텍스트 점검은 실제 `<img>` 요소와 링크 텍스트, FAQ 버튼 접근명 기준으로 봅니다.
- Best Practices `96`은 로컬 `/_vercel/*` 404 기록과 실제 코드 이슈를 분리해서 해석합니다.

## 2. WCAG 2.1 AA 후속 후보

### 2-1. Blog placeholder 링크 상태 정리

현재 상태:
- `#blog` 카드 3개가 모두 `href="#blog"` 자기참조 링크입니다.
- 시각적으로는 "오픈 예정"이지만, 키보드와 스크린리더에서는 실제 이동 가능한 링크로 노출됩니다.

후속 액션:
- 실제 URL이 없을 때는 링크 대신 비활성 카드 또는 버튼이 아닌 정적 요소로 전환하는 안을 검토합니다.
- 최소 수정으로는 `aria-disabled="true"` 성격의 표현과 실제 이동 차단 구조를 정리하는 것이 좋습니다.

예상 임팩트:
- placeholder 콘텐츠에서 생기는 접근성 혼선을 줄일 수 있습니다. [즉시 수정 가능]

### 2-2. 모바일 메뉴 대화상자 회귀 점검 유지

현재 상태:
- `nav__mobile`은 `role="dialog"`와 focus trap을 갖고 있고, Escape/Tab 순환도 구현돼 있습니다.
- 현재 점검에서는 이상이 없었습니다.

후속 액션:
- nav 구조를 다시 바꾸는 PR이 나오면 iOS Safari, Android Chrome 기준으로 포커스 이동과 닫기 동작을 재확인합니다.

예상 임팩트:
- 모바일 메뉴 접근성 회귀를 조기에 막을 수 있습니다. [즉시 수정 가능]

### 2-3. Hero/OG 이미지 교체 시 대비 재측정

현재 상태:
- Hero 텍스트는 현재 배경 사진 기준으로 가독성이 확보돼 있습니다.
- 다만 Hero 배경 사진은 이후 교체 가능성이 높습니다.

후속 액션:
- 새 Hero 사진이 들어오면 제목, CTA, 스크롤 인디케이터 대비를 다시 측정합니다.
- 특히 좌하단 텍스트 영역이 밝아지면 overlay 강도 조정이 필요할 수 있습니다.

예상 임팩트:
- WCAG 대비 저하를 사진 교체 단계에서 바로 막을 수 있습니다. [콘텐츠 입력 대기]

### 2-4. 후기 실데이터 반영 시 문장 길이 점검

현재 상태:
- 후기 4건은 현재 placeholder이며 blockquote/footer 구조는 적절합니다.

후속 액션:
- 실제 후기 입력 시 한 카드 안에 1~3줄 범위를 유지하고, 너무 긴 문장이 모바일에서 카드 높이를 과도하게 늘리지 않는지 확인합니다.

예상 임팩트:
- 읽기 흐름과 모바일 스캔성이 유지됩니다. [콘텐츠 입력 대기]

## 3. SEO 후속 후보

### 3-1. OG 전용 이미지 제작 및 메타 보강

현재 상태:
- `og:image`, `twitter:image`는 현재 로고 파일을 사용합니다.
- head 주석상 1200x630 전용 OG 이미지는 아직 placeholder 상태입니다.

후속 액션:
- 전용 OG 이미지 제작 후 `og:image`, `twitter:image`를 교체합니다.
- 함께 넣을 메타:
  - `og:image:alt`
  - `twitter:image:alt`
  - 필요 시 `og:site_name`

예상 임팩트:
- 카카오톡/메신저/소셜 공유 카드 품질이 즉시 좋아집니다. [콘텐츠 입력 대기]

### 3-2. sitemap `lastmod` 갱신 기준 정리

현재 상태:
- `sitemap.xml`의 `lastmod`는 `2026-04-05`로 고정돼 있습니다.
- 현재 로컬 main은 그 이후 변경을 포함합니다.

후속 액션:
- 다음 push 또는 배포 시점에 맞춰 `lastmod`를 실제 반영 날짜로 갱신합니다.
- 수동 유지가 번거로우면 배포 전 갱신 규칙도 추후 검토할 수 있습니다.

예상 임팩트:
- 검색엔진에 최신 반영 시점을 더 명확히 전달할 수 있습니다. [즉시 수정 가능]

### 3-3. `WebSite` 구조화 데이터 추가 검토

현재 상태:
- 현재는 `LocalBusiness`와 `FAQPage`만 들어 있습니다.

후속 액션:
- 단일 랜딩 구조에 맞는 `WebSite` 스키마를 추가해 사이트 정체성과 canonical 문맥을 보강할지 검토합니다.
- 검색 기능이 없으므로 `SearchAction`은 불필요합니다.

예상 임팩트:
- 구조화 데이터의 맥락이 조금 더 풍부해집니다. [즉시 수정 가능]

### 3-4. Naver Place / 외부 프로필 `sameAs` 확장

현재 상태:
- `sameAs`에는 카카오톡 오픈채팅, 인스타그램만 있습니다.
- 네이버플레이스 연결은 아직 확정되지 않았습니다.

후속 액션:
- 공식 Naver Place URL이 확정되면 `sameAs` 추가를 검토합니다.

예상 임팩트:
- 로컬 비즈니스 신뢰도와 검색 문맥을 보강할 수 있습니다. [콘텐츠 입력 대기]

## 4. Best Practices 96 → 100 해석

### 4-1. 로컬 전용 기록

현재 상태:
- Lighthouse `Best Practices 96`의 직접 원인은 로컬 서버에서만 발생한 `/_vercel/insights/script.js`, `/_vercel/speed-insights/script.js` 404입니다.
- 이는 `python3 -m http.server` 환경에서 Vercel 전용 스크립트를 서빙하지 않기 때문에 생긴 기록입니다.

판단:
- production Vercel 환경과 직접 연결된 코드 결함으로 보지 않습니다.
- push 이후 실제 배포 URL에서 재측정해 동일 현상이 사라지는지 먼저 확인하는 것이 우선입니다.

예상 임팩트:
- 불필요한 코드 수정 없이 원인을 정확히 분리할 수 있습니다. [즉시 수정 가능]

### 4-2. production 재측정 체크

후속 액션:
- push 차단 해소 후 production URL에서 Lighthouse를 다시 측정합니다.
- production에서도 BP가 100이 아니면 아래 순서로 확인합니다.
  1. Vercel Analytics / Speed Insights 스크립트 정상 응답 여부
  2. 외부 링크 target/rel 조합 이상 여부
  3. 콘솔 네트워크 오류 유무

예상 임팩트:
- 로컬 잡음을 제외한 실제 후속 수정 대상을 빠르게 좁힐 수 있습니다. [즉시 수정 가능]
