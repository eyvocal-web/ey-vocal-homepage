# EY보컬스튜디오 Blog Placeholder 링크 분석

대상 프로젝트: `ey-vocal-homepage`

현재 상태:
- `#blog` 카드 3건이 모두 `href="#blog"` 자기참조 링크입니다.
- 시각적으로는 "오픈 예정" 카드지만, 키보드와 스크린리더에는 이동 가능한 링크로 읽힙니다.

## 1. 옵션 비교

### 1순위: 링크 제거 후 정적 카드로 전환

방법:
- `<a class="blog-card__link">`를 링크가 아닌 정적 래퍼 요소로 바꿉니다.
- 카드 안의 텍스트와 badge만 유지합니다.

장점:
- 실제 이동이 없는 placeholder 상태를 가장 정확하게 표현합니다.
- 스크린리더와 키보드 사용자에게 불필요한 포커스 대상을 줄일 수 있습니다.
- 현재 Blog 카드의 "오픈 예정" 의미와도 가장 잘 맞습니다.

리스크:
- 카드 전체 클릭 UX는 사라지지만, 실제 링크가 없는 상태라 기능 손실은 아닙니다.

### 2순위: 링크 유지 + `aria-disabled` 계열 표현

방법:
- 링크를 유지하되 비활성 상태 의미를 추가로 전달합니다.

장점:
- 마크업 변경 폭이 더 작을 수 있습니다.

단점:
- `<a>`는 본질적으로 이동 요소라, 실제로는 클릭되지 않는 링크를 남겨 두면 의미가 덜 깔끔합니다.
- 키보드 포커스는 계속 남아 혼선이 완전히 사라지지 않습니다.

## 2. 현재 권장안

- 1순위: 링크 제거한 정적 카드 전환
- 2순위: 불가피할 때만 `aria-disabled` 계열 표현 검토

판단 근거:
- 현재 목적은 실제 글 링크 제공이 아니라 placeholder 안내입니다.
- 접근성 측면에서는 "기능 없는 링크"를 남겨 두지 않는 편이 더 명확합니다.

## 3. 회귀 리스크

WCAG 2.1 AA:
- 1순위 안이 더 안전합니다.
- 포커스 순서와 링크 텍스트 혼선을 줄일 수 있습니다.

Lighthouse:
- 현재 `100 / 100 / 96 / 100` 기준에서 회귀 리스크는 낮습니다.
- 오히려 접근성/Best Practices 문맥에서는 개선 가능성이 있습니다.

nav focus trap:
- Blog 카드 수정은 `#blog` 섹션 내부 마크업만 건드리는 작업이라 모바일 메뉴 focus trap과는 직접 연결되지 않습니다.
- `js/main.js`의 nav 로직과도 별개입니다.
