# EY보컬스튜디오 분석 도구 비교 메모

대상:
- 정적 HTML/CSS/JS 사이트
- 현재 원칙: 외부 런타임 의존성 최소화

## 1. 결론 요약

현재 기준 우선순위:
1. Vercel Analytics 검토
2. 필요 시 GA4는 성표님 예외 승인 + Measurement ID 확보 후 별도 적용

이유:
- 지금 저장소 규칙상 외부 CDN 스크립트 추가는 기본 금지입니다.
- GA4는 일반적으로 외부 `gtag.js` 로드가 필요합니다.
- 따라서 GA4는 바로 삽입보다 “조건 충족 시 후속 적용”이 안전합니다.

## 2. 비교표

| 항목 | Vercel Analytics | GA4 |
|---|---|---|
| 적용 성격 | Vercel 생태계 중심 | Google 분석 플랫폼 |
| 구현 복잡도 | 상대적으로 낮음 | 상대적으로 높음 |
| 외부 스크립트 부담 | 낮은 편 | 높음 (`gtag.js`) |
| 커스텀 이벤트 확장 | 제한적일 수 있음 | 풍부함 |
| 마케팅/광고 연계 | 제한적 | 강함 |
| 현재 원칙 적합성 | 높음 | 예외 승인 필요 |

## 3. Vercel Analytics를 먼저 보는 이유

- 현재 프로젝트가 Vercel 배포를 전제로 함
- 정적 사이트에 맞게 최소 설정으로 시작하기 쉬움
- 페이지 조회, 기본 유입 확인부터 시작하기 적합

권장 상황:
- 먼저 전체 트래픽과 기본 성능 흐름을 보고 싶을 때
- 코드 변경을 최소화하고 싶을 때

## 4. GA4를 나중으로 미루는 이유

- 일반적으로 외부 스크립트 로드가 필요함
- Measurement ID가 있어야 실제 삽입 가능
- 이벤트 설계를 하지 않으면 단순 삽입만으로 실효성이 낮음

현재 적용 전 필수 조건:
- 성표님이 GA4 적용을 명시적으로 승인
- Measurement ID 제공
- 외부 스크립트 예외 승인
- 추적 이벤트 범위 합의

## 5. GA4 적용 시 필요한 것

### 필수 값

- GA4 Measurement ID
  - 형태 예시: `G-XXXXXXXXXX`

### 기본 삽입 위치

- `index.html`의 `<head>`

### 적용 전 결정할 것

- 페이지뷰만 볼지
- 카카오톡 상담 클릭 추적까지 할지
- 전화 링크 클릭도 추적할지
- 스크롤/FAQ 열람 같은 세부 이벤트까지 볼지

## 6. 추적 우선순위 제안

1단계:
- 페이지 조회
- 카카오톡 상담 버튼 클릭
- 전화 링크 클릭

2단계:
- FAQ 열람
- 주요 섹션 도달
- 모바일/데스크톱 분기 확인

## 7. 권장 진행 순서

1. Vercel Analytics 가능 범위 확인
2. 대표 도메인 연결
3. Search Console 등록
4. 기본 유입 추이 확인
5. 필요성이 확인되면 GA4 예외 승인 후 적용

## 8. 성표님 확인 필요 항목

- Vercel Analytics만 먼저 볼지
- GA4까지 바로 갈지
- GA4를 원하면 Measurement ID 발급 여부
- 추적하고 싶은 핵심 이벤트 3개 선정
