# EY보컬스튜디오 홈페이지 — IA & UX 전략 문서

## 1. 프로젝트 개요

- **비즈니스**: EY보컬스튜디오 — 이천 소재 프리미엄 1:1 보컬 레슨 스튜디오
- **사이트 유형**: 단일 페이지(One-page) 랜딩 홈페이지
- **목표 출시**: 정적 HTML/CSS/JS (외부 의존성 없음)

---

## 2. 타겟 오디언스

| 세그먼트 | 특성 | 핵심 니즈 |
|---------|------|----------|
| **입시생 (고2~재수)** | 실기 준비, 부모가 의사결정자 | 합격 실적, 체계적 커리큘럼, 신뢰 |
| **취미 성인 (20~40대)** | 직장인/대학생, 노래를 잘 하고 싶음 | 부담 없는 분위기, 눈에 보이는 실력 변화 |
| **전문 보컬리스트/유튜버** | 디테일한 테크닉 보완 | 전문성, 장비 수준, 레코딩 가능 여부 |
| **학부모** | 자녀 입시 대리 탐색 | 강사 경력, 수업 환경, 접근성(이천 내) |

**1차 타겟**: 입시생 + 학부모 (전환 가치 높음)
**2차 타겟**: 취미 성인 (볼륨 확보)

---

## 3. 페이지 목표 (우선순위순)

1. **신뢰 구축** — "이 선생님에게 배우면 되겠다"는 확신
2. **상담 전환** — 카카오톡 / 전화 상담 연결
3. **차별화 인식** — 대형 학원이 아닌 1:1 프리미엄의 가치 전달
4. **위치 인지** — 이천에서 쉽게 찾아올 수 있음을 보여줌

---

## 4. 사이트 구조 (섹션 순서)

단일 페이지, 스크롤 기반. 사용자 심리 흐름에 맞춰 배치:

```
[히어로] → 관심 끌기
  ↓
[스튜디오 소개] → 신뢰 시작
  ↓
[강사 프로필] → 전문성 확인
  ↓
[레슨 프로그램] → "나에게 맞는 수업이 있네"
  ↓
[수강생 후기] → 사회적 증거
  ↓
[시설 갤러리] → 환경 확인
  ↓
[위치 & 오시는 길] → 물리적 접근성
  ↓
[상담 신청 / CTA] → 전환
  ↓
[푸터] → 부가 정보
```

총 **9개 섹션**, 스크롤 깊이 약 5~7 뷰포트.

## 5. 섹션별 상세 설계

### 5.1 히어로 (Hero)

- **구성**: 풀스크린 배경 (스튜디오 내부 또는 레슨 장면) + 헤드카피 + 서브카피 + CTA 버튼
- **헤드카피 방향**: 결과 중심. 예) "당신의 목소리, 달라질 수 있습니다"
- **CTA**: `무료 상담 신청하기` (카카오톡 연결)
- **UX 결정**: 자동 재생 영상 대신 정적 이미지 + 오버레이. 모바일 데이터 절약, 로딩 속도 우선.

### 5.2 스튜디오 소개 (About)

- 2~3문장으로 핵심 가치 전달
- 키워드: `1:1 맞춤`, `이천`, `프리미엄`, `체계적`
- 아이콘 3개 정도로 차별점 요약 (예: 1:1 레슨 / 맞춤 커리큘럼 / 녹음 피드백)

### 5.3 강사 프로필 (Instructor)

- 사진 + 이름 + 한 줄 소개
- 주요 경력 3~5줄 (학력, 활동, 지도 실적)
- **UX 결정**: 너무 긴 이력은 접이식(accordion)으로 처리. 첫인상은 간결하게.

### 5.4 레슨 프로그램 (Programs)

- 카드 UI, 2~3개 프로그램
  - **입시반**: 대상, 내용, 주 횟수
  - **취미반**: 대상, 내용, 주 횟수
  - **전문반/특별 과정** (선택)
- 가격은 명시하지 않음 → 상담 유도 전략
- 각 카드 하단에 `자세히 상담받기` 마이크로 CTA

### 5.5 수강생 후기 (Testimonials)

- 카드 슬라이더 또는 세로 나열 (3~5개)
- 이름(이니셜) + 수강 과정 + 후기 텍스트
- **UX 결정**: 실명보다 이니셜 + 과정명이 신뢰와 개인정보 균형에 적합

### 5.6 시설 갤러리 (Gallery)

- 그리드 레이아웃, 4~6장
- 레슨룸, 장비, 녹음 부스 등
- 라이트박스(클릭 시 확대) — JS로 직접 구현 (외부 의존성 없음)

### 5.7 위치 & 오시는 길 (Location)

- 주소 텍스트 + 간단한 안내
- 지도: 카카오맵 또는 네이버 지도 정적 이미지 + 링크 (iframe embed 최소화)
- 주차 안내, 대중교통 한 줄

### 5.8 상담 신청 (Contact CTA)

- 큰 CTA 블록: 배경색 강조
- 카카오톡 채널 버튼 (주 전환 채널)
- 전화번호 (클릭 시 `tel:` 링크)
- 운영 시간 표기

### 5.9 푸터 (Footer)

- 상호명, 대표자, 사업자번호, 주소
- 인스타그램 / 유튜브 링크 (있을 경우)
- Copyright

## 6. CTA 전략

| 위치 | CTA 텍스트 | 유형 | 목적 |
|------|-----------|------|------|
| 히어로 | 무료 상담 신청하기 | Primary 버튼 | 첫 전환 시도 |
| 레슨 프로그램 카드 | 자세히 상담받기 | Ghost/텍스트 링크 | 관심 세분화 |
| 플로팅 | 카카오톡 상담 | FAB (우하단) | 어디서든 접근 |
| 상담 섹션 | 지금 바로 상담하기 | Primary 버튼 (큰) | 최종 전환 |
| 네비게이션 | 상담 신청 | Primary 버튼 (작은) | 상시 노출 |

**플로팅 카카오톡 버튼**: 스크롤 시 항상 우하단 고정. 모바일에서 특히 중요.

## 7. UX 핵심 결정

### 네비게이션
- **데스크톱**: 상단 고정(sticky) 내비. 로고 좌측, 섹션 링크 중앙, CTA 우측.
- **모바일**: 햄버거 메뉴. 열면 풀스크린 오버레이.
- 스크롤 시 네비 배경 불투명도 변경 (스크롤 인식)

### 반응형 전략
- **모바일 퍼스트** 설계. 이천 지역 사용자 대부분 모바일 접근 예상.
- 브레이크포인트: 768px (태블릿), 1024px (데스크톱)
- 이미지: 모바일용 경량 버전 별도 제공 (`<picture>` + `srcset`)

### 성능
- 외부 라이브러리 제로. 순수 HTML/CSS/JS.
- 이미지 lazy loading (`loading="lazy"`)
- 폰트: 시스템 폰트 스택 우선, 필요 시 Pretendard (self-host)
- 목표: Lighthouse 성능 90+

### 접근성
- 시맨틱 HTML (`<header>`, `<nav>`, `<main>`, `<section>`, `<footer>`)
- 충분한 색상 대비 (WCAG AA)
- 키보드 내비게이션 지원
- `alt` 텍스트 필수

### 인터랙션
- 과도한 애니메이션 지양. 스크롤 시 페이드인 정도만.
- 부드러운 스크롤 (`scroll-behavior: smooth`)
- 갤러리 라이트박스만 JS 필수, 나머지는 CSS 우선

## 8. 한국어 콘텐츠 톤 & 보이스

### 톤 가이드라인

| 항목 | 방향 |
|------|------|
| **존칭** | 합쇼체 (ㅂ니다/습니다) 기본. 부드럽지만 격식 있게. |
| **문장 길이** | 짧게. 한 문장 20자 내외 목표. |
| **감성 vs 정보** | 히어로·후기는 감성적, 프로그램·소개는 정보 중심 |
| **과장 금지** | "최고", "독보적" 같은 과장 표현 지양. 구체적 실적으로 대체. |
| **전문 용어** | 보컬 용어는 괄호 설명 병기. 예) 믹스보이스(혼합 발성) |
| **지역성** | "이천"을 자연스럽게 노출. 지역 밀착 이미지. |

### 톤 예시

```
❌ "대한민국 최고의 보컬 트레이닝을 경험하세요!"
✅ "1:1 맞춤 레슨으로, 당신만의 목소리를 찾아드립니다."

❌ "저희 학원은..."
✅ "EY보컬스튜디오는..."

❌ "수강료 문의 환영합니다~"
✅ "레슨 상담은 카카오톡으로 편하게 문의해 주세요."
```
